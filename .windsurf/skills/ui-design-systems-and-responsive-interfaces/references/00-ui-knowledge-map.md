## Quick Sequence

1. Define constraints and success metrics.
2. Choose layout and component strategy.
3. Define CTA hierarchy and action labeling rules.
4. Specify responsive behavior and accessibility rules.
5. Define token and component governance.
6. If the work is brownfield or component-heavy, capture redesign boundaries and component verification loops.
7. Use the local generator when you need a structured first-pass packet or persistence-safe master and override artifacts.
8. Run authenticity benchmark and anti-generic style checks.
9. Define testing and quality gates.
